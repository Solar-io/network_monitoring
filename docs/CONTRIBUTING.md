# Contributing

Branching
- Trunk-based: `main` protected; short-lived `feature/<scope>` branches; squash merges.

Commit Style
- Conventional commits: `feat|fix|docs|chore|refactor|test(scope): summary`.

PR Checklist
- [ ] Minimal scope; linked issue/ADR/STATUS section
- [ ] Docs updated (STATUS/TASKS/BUGLOG/EXPERIMENTS if relevant)
- [ ] Tests/evals run; screenshots for UI
- [ ] Snapshot created before risky changes

Code Style
- TypeScript strict; ESLint + Prettier; Tailwind plugin; import order enforced.

Reviews
- Seek early feedback with Draft PRs; respond with deltas and test evidence.
